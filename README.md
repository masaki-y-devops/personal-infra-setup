# personal-infra-setup
独学でのPC・サーバー環境構築で詰まったところ、解決策の記録。

## 設定を書き始めたきっかけ
小学生くらいのころに親のお下がりで古いノートパソコンをもらい、Windowsが厳しかったので書店でインストールDVD付きの書籍を買ってもらい、Ubuntu（当時は8.04）あたりを触り始めたのが最初のきっかけ。

当時はまったく使い方がわからず、標準環境でメモやブラウザを動かすだけだった。

大学時代には、東京に下宿していたこともあり秋葉原に足しげく通った。中古ノートパソコンを買っては色々なディストロを入れて楽しんでいた。このころからArchの面白さに触れていく。

大学院時代には、NixOSやGentooに加え、自分でWebサービスを動かしてみたいと思い、サーバー環境構築にもトライした。

小さいころに祖父からもらったPDAや電子辞書のお下がりを使うのが好きで、そのつながりで超小型PC好きになった。GPD製品をはじめとするUMPCで自分の設定が活きると楽しく、それがモチベーションになっている。

## Desktop
### Arch
Manjaroライブ環境からinstall.sh実行で自分環境構築を完了するように。
bootstrapでインストール用環境を一時展開した後、インストール先特定。
既存パーティション削除、再作成→インストール→ブートローダー再設定→設定済みi3wm＋個人アプリ導入まで自動化。

#### 主に詰まったところ
1. 異なるディストロのライブ環境からのインストールに挑戦してみたところ、bootstrapで一時環境の構築が必要だった。
その際の各種マウントやpacmanの初回処理が慣れておらず、日本語と英語のArchWikiを両方読み、実験用のUMPCの環境を何度もぶっ壊しながら、問題なく動く方法を探った。
2. 設定ファイル書き出しの際の変数展開の有無の制御。
3. ユーザーファイルの権限問題。
i3,i3statusの設定ファイルをほかのシステム設定ファイルと同様に書き込むと、所有者がrootとなり、そのままだとユーザーでログインしたときに設定ファイルが読めない。
これらを見落としており、はまった。

### NixOS
Manjaroライブ環境からinstall.sh実行で自分環境構築を完了するように。

#### 主に詰まったところ
1. Nix言語の書式問題。「結果」から記述していくスタイルは全く初めてで楽しめた。
その反面、関数の選定ミスや記述ミスに加え、コンマの抜けによるコンパイルエラーが多発し、時間を費やした。
また、Archよりもデフォルトの設定にクセが強いと感じることもあり、自分的によりプレーンな状態にもっていくのがたいへんだったと思う。
2. インストールスクリプトの書き方に詰まった。
関数型OSという慣れない環境特有のはまりポイントだったかもしれない。
```installer.sh```をヒアドキュメントで書き出し、nixパッケージマネージャーをセットアップしたうえで、それを実行。大まかな実行内容としては、```configuration.nix```を編集、```nixos-install```を実行するというシンプルな流れだが、これを見つけるのに手間取った。愚直に公式サイトやドキュメントをよく読み、組み立てればうまくいきやすいことを学んだ。

### Gentoo
設定経験はあるが、コンパイルに時間がかかりすぎるのでトライアンドエラーが十分に回せず、自動化はしていない。

### Mint
使わないプリインパッケージの削除、個人的に必須なアプリのインストール＋PPA追加。
ArchやNixOSでも発生する問題の応急処置的なFixを自動化（GPD製UMPCの画面回転問題、タッチパネル座標関連、Intel系サウンドデバイスの終了時ポップノイズ抑制など）。

#### 主に詰まったところ
Ubuntuベースで、"out-of-the-box"なディストロということもあり、インストール自体は手間取ることはなかった。
ただ、ArchやGentoo,NixOSが「ゼロから足していく」アプローチを取るのに対し、Mint、MXLinuxなどは自分好みの設定にするにはすでにあるものを「引き算」していくことになるので、まずデフォルトの設定を大まかに理解して、消す、変えるものとそのままにしておくものを選別するのが重要なポイントだと感じた。
画面回転などの設定は、Archなどのものと共通店が多く、部分的にそれを適用するだけで済んだので、元々の知識で何とかなった。

### Windows
使わないUWPアプリの削除、レジストリ操作、手動更新が必要なアプリ（7-Zip,rclone,mpvなど）の起動時更新タスクの追加。

## 使用例

本スクリプトをタスクスケジューラに登録してログイン時に実行します。

クラウドストレージ上にスクリプトを置き、同期クライアントの起動を待って（3分と想定）実行する場合:

~~~
schtasks /create /tn UpdateUserApps /tr "K:\Scripts\Windows\Tasks\UpdateUserApps.bat" /sc onlogon /delay 0003:00 /f
~~~

## 作成経緯

Windows環境において、付属の自動更新機能がない、あるいは独自にバイナリを管理したいツール（rclone, 7-zip等）があります。

その更新作業は手動に頼らざるを得ず、「更新漏れによるセキュリティリスク」と「毎回繰り返しのチェック作業の煩雑さ」が課題でした。

本スクリプトは、これらのツールを一括で、できるだけ最新の状態に保つために作成しました。

## 選定理由

確かに7-zipやrcloneはchocolateyパッケージがあります。

しかし、パッケージング作業という中間工程がある関係上、公式サイトやGitHubから直接実行ファイルをダウンロードするよりも不確実性リスクが大きいと判断しています。

また、chocolateyやwingetにはサービス終了のリスクもゼロではないと考え、長期運用には不安が残る可能性があるため、上流から直接引っ張ってくる選択肢を選びました。

## トラブルシューティング

### WSLのAPTコマンド実行において、事前に```/etc/sudoers```の編集が必要（```visudo```コマンド使用）。

Windows再インストール後走らせたところ、スクリプトがsudoのパスワード待ち受け状態で止まっていたため認識。

APTコマンドはパスワードなしで実行できてしまうので、セキュリティー的には問題ありの可能性高し。代替を検討中です。

## 今後の課題

### 冪等性

現状でも各ソフトウェアがインストール済みでない場合は処理をスキップするが、rclone,mpvの処理では外部スクリプトに頼っています。

現状のままだと、外部スクリプトがエラーでも処理が流れて正常終了となってしまうのが要検討事項です。

### エラーハンドリング

エラーの場合に、ユーザーの介入が必要となります。

特に7-Zipの処理で、7-zip.orgと通信できなかった場合と、サイト構成の変化により、スクレイピングでバージョン抽出できないとき、問題が顕在化します。

このあたりの落としどころは要検討事項となっています。

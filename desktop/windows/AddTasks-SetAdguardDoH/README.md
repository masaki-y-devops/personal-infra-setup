## 作成の経緯

インターネット上の広告は人々の需要と供給をつなぎ、社会に価値を提供しているのは間違いありません。

しかし、近年、悪質なフィッシング広告やマルウェアを含んだサイトに誘導する広告が増えています。

そのため、エンドユーザーとしては、そのような広告をブロックすることで、自分のパソコンのセキュリティを維持する必要も生じているのが実情と考えます。

その実践として、Windows上のDNS over HTTPS設定をワンストップで作成するスクリプトを書いてみました。

## 課題点

システム上のすべてのDNS名前解決に対し、AdguardパブリックDNSを使用する設定となっています。

そのため、プロバイダ提供のDNSサーバーではない関係上、アクセスできないサイトが生じたり、サイトレイアウトが崩れる可能性が常にあります。

また、AdGuard社のシステム障害にも影響を受けてしまいます。

そのため、フォールバック先としてControl-D社のDNSサービスを検討していますが、AdGuard社のサービスが安定しているので、同社のアドレスを指定しています。
